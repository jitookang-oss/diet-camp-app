## 개발 원칙

> 코딩 시 모든 UI/UX 결정의 기준이 되는 원칙이다. 기능 추가·수정 시 반드시 이 원칙에 맞는지 확인할 것.

1. **참여자가 쉽게 사용할 수 있는 UI를 전제로 만든다.**
   - 비개발자, 중장년층 포함 다양한 연령대가 사용함
   - 입력 단계는 최소화, 화면당 목적은 하나로 명확하게
   - 혼란스러운 분기·에러보다 자연스러운 안내 흐름 우선

2. **다이어트 동기부여를 받을 수 있도록 유도한다.**

   - 칭찬·격려 문구로 긍정적 피드백 제공

## 화면 구성 (페이지별)

### `/` — 메인 홈
- D-day 카운터: 시작 전 D-N / 진행 중 N일차·N주차 + 진행률 바 / 종료 후 완료 배지
- 이번 주 미션 카드 (해당 주차 자동 표시) → 클릭 시 미션 상세 모달
- **기존 참여자:** "이어서 하기" 버튼 (대시보드로 이동)
- **신규 참여자:** 인스타그램 아이디(@없이) + 생년월일 입력 → 온보딩으로 이동
- 기존 데이터 있으면 등록 폼 숨김 (중복 등록 방지)

### `/invite` — 초대 링크 진입
- 관리자가 생성한 초대 링크로만 캠프 참여 가능
- URL 파라미터: `phone` (사전 등록된 전화번호)
- 검증 후 `/onboarding`으로 이동, `invite_phone`을 localStorage에 저장

### `/onboarding` — 기본정보 입력 (4단계: Step 0~3)
- **Step 0:** 1회차 시즌 안내 + @bora__magic 인스타그램 DM 링크 + 개인정보 동의 (필수) — 항상 표시
- **Step 1:** 성별, 키, 몸무게 → BMI·목표체중 자동계산 (현재 × 0.85)
- **Step 2:** 카카오톡 알림 수신 번호 **(필수)**, 복용약 여부(+상세), 질환 여부(+상세)
- **Step 3:** 호르몬 변화 관련 증상 (여성만, 최근 6개월 기준 복수 선택) → 완료 시 `/survey`로 이동
  - 항목: 생리 주기 불규칙, 얼굴 홍조/열감, 수면 질 저하/식은땀, 감정 기복, 체중·복부지방 변화, 피부 건조·탈력·탈모, 피로·무기력

### `/survey` — 20문항 설문 (`?week=N` 파라미터)
- `week=1`: 초기 설문 → 체질 유형 분석 후 `/result`로
- `week=2~11`: 주차별 정기 설문 → 완료 시 몸무게 입력 → `/dashboard`로
- `week=12`: 최종 설문 → `/result?week=12`로

### `/result` — 결과 페이지
- 체질 유형 배지 + 설명 + 핵심 가이드
- 대사관리 스코어 바 (음식/활동/멘탈, 100점 만점) + 레이더 차트
- 12주 전후 비교 (최종 설문 완료 시)
- 약사 추천 영양제 (1주차만, 체질 유형별 맞춤 4그룹)
- PDF 결과지 다운로드 (html2canvas + jsPDF, A4)
- "매주 기록하러 가기" 또는 "최종 대시보드 보기" 버튼

### `/dashboard` — 참여자 대시보드
- 즐겨찾기 유도 배너 (첫 방문 시만 표시)
- 이번 주 미션 카드 → 클릭 시 미션 상세 모달 오픈
- **이번 주 미션 현황 카드:** 월~일 요일별 달성 체크 (탭으로 토글, API 저장)
- 몸무게 + 목표 달성률 요약 카드
- 체질 유형 카드
- 몸무게 변화 LineChart (목표 기준선 포함)
- 대사관리 스코어 변화 LineChart
- 주차별 기록 테이블
- **주차별 기록 버튼:** 날짜 잠금 (1주차 기록하기 → 6/7~, 매주 7일 간격)
  - 잠금 시: 🔒 표시 + 해제 날짜 안내
  - 해제 시: 버튼 활성화 → `/weekly` 경유 → `/survey?week=N`으로

### `/weekly` — 주차 라우팅 (redirect only)
- 날짜 잠금 체크 → 미해제 시 `/dashboard`로 돌려보냄
- 해제됐으면 적절한 `/survey?week=N`으로 이동

### `/checkin` — 체크인 페이지 (개편 예정)

> ⚠️ 기존 URL 파라미터 방식(token/phone/date/type)에서 **자기기록형 체크리스트**로 전면 개편 예정.  
> 알림톡 버튼 URL은 `https://diet-camp-app.vercel.app/checkin` 고정 — 템플릿 변경 불필요.

**사용자 식별:** localStorage의 phone → 자동 식별. 없으면 인스타 아이디 입력 유도.

**섹션 1 — 오늘의 체크리스트 (4항목)**
- ① 아침 영양제: 본인이 저장한 영양제 이름 표시. 미입력 시 "영양제 정보 입력하기" 유도
- ② 점심 걷기
- ③ 저녁 운동스낵
- ④ 이번 주 미션: 해당 주차 미션명 표시 (예: "식후 10분 안에 10분 걷기")
- 체크/해제 → `daily_checkins` 테이블 저장 (기존 supplement/lunch_walk/evening_exercise/mission 타입 활용)

**섹션 2 — 이번 주 달성률 (4항목 개별)**
- 영양제 N/7 · 걷기 N/7 · 운동 N/7 · 미션 N/7 형태로 각각 표시
- 동기부여 시각화 목적

**섹션 3 — 주간 기록 알림 (조건부)**
- 이번 주 설문·몸무게 미입력 시에만 표시: "N주차 기록이 아직 없어요 → 지금 하기" (→ `/weekly`)
- 완료 시 숨김

**영양제 정보 저장:**
- `participants.my_supplements` (text) 컬럼 신규 추가
- 한 번 저장하면 유지, 수정 버튼으로 편집 가능

### `/admin` — 관리자 페이지 (약사 전용)
- 비밀번호 로그인 (`NEXT_PUBLIC_ADMIN_PASSWORD`)
- **참여자 목록 탭:**
  - 전체 참여자 테이블 (인스타 아이디, BMI, 체질유형, 초기점수, 감량, 진행주차)
  - 참여자 클릭 → 간략 모달 (초대 링크, 신체 정보, 건강 상태, 대사관리 점수 비교, 주차별 몸무게)
  - **상세 조회 버튼** (1주차 설문 완료 또는 호르몬 증상 데이터 있는 참여자에 한해 표시):
    - 몸무게 변화 LineChart
    - 대사관리 스코어 변화 LineChart (음식/활동/멘탈/종합)
    - 레이더 차트 (최신 주차 기준)
    - 설문 답변 상세 (1주차 vs 12주차 문항별 비교, PART별 펼치기)
    - 호르몬 변화 관련 증상 목록
  - 참여자 추가 버튼 (이름+전화번호 단건 등록)
  - 엑셀/CSV 일괄 업로드 (xlsx 파싱 → 미리보기 → 등록)
  - 초대 링크 복사 버튼 (참여자별)
  - 참여자 삭제 버튼
- **체크인 현황 탭:**
  - 날짜 선택 → 당일 체크인 내역 조회 (영양제/점심걷기/저녁운동/주간미션)
  - **주간 요약 뷰 (신규 추가 예정):** 참여자(행) × 4항목(열) 형태로 이번 주 달성률 한눈에 파악

### `/reset` — 데이터 초기화 페이지
- 이 기기의 localStorage 전체 삭제
- 초대 링크로 다시 시작할 수 있도록 안내

## 카카오 알림톡 시스템

### 발송 일정 (Vercel Cron, UTC 기준)

| 시간(KST) | UTC | 내용 | 체크인 타입 |
|-----------|-----|------|-------------|
| 오전 8시 | 23:00 전일 | 영양제 챙기기 | `supplement` |
| 낮 12:30 | 03:30 | 점심 걷기 | `lunch_walk` |
| 저녁 7시 | 10:00 | 저녁 운동 | `evening_exercise` |

> ⚠️ **현재 cron 중단 중** (2026-06-08~): 체크인 링크 방식 개편 작업 전까지 자동 발송 비활성화. `vercel.json`의 `crons` 배열이 비어 있음.

### 보안
- 크론 API: `Authorization: Bearer {CRON_SECRET}` 헤더 검증
- 체크인 링크: HMAC-SHA256(`phone:date:type` + `CHECKIN_SECRET`) 앞 16자리
- 중복 방지: Supabase `unique(phone, check_date, type)` 제약

### 체크인 링크 개편 계획 (미구현)

**현재 문제:**
- 솔라피 알림톡 템플릿에 버튼 URL이 고정값(`https://diet-camp-app.vercel.app/checkin`)으로 등록되어 있음
- 코드에서 사용자별 동적 URL을 생성해서 넘기고 있지만, 템플릿 고정값에 의해 무시됨

**확정된 방향 (방향 A — 사용자별 고유 토큰):**
- 참여자 테이블에 `checkin_token` 컬럼 추가 (UUID, 온보딩 시 발급, 영구 고정)
- 알림톡 버튼 URL: `https://diet-camp-app.vercel.app/checkin?u={checkin_token}`
- 체크인 페이지: 토큰으로 참여자 조회 → 현재 KST 시간으로 체크인 타입 자동 판별 → DB 저장
- 솔라피 템플릿 버튼 URL을 변수(`#{체크인링크}`)로 재등록 필요 (카카오 재검수 필요할 수 있음)

**코딩 전 확인 필요:**
1. 솔라피 대시보드에서 현재 템플릿 버튼 URL 방식 확인 (고정 vs 변수)
2. 템플릿 재등록 시 카카오 재검수 필요 여부 확인

#### 1주차 (6/1–6/7): 커피는 하루 1잔, 식후 2시간 후에
> **상담 주간:** 인스타그램 DM으로 개별 연락 예정

카페인은 우리 몸의 교감신경을 자극합니다. 교감신경이 활성화되면 몸은 '긴장 모드'로 전환되어 지방을 저장하려는 방향으로 호르몬이 분비됩니다. 특히 공복 상태에서 커피를 마시면 코르티솔(스트레스 호르몬) 수치가 급격히 올라가고, 위산 분비가 자극되어 속이 쓰리고 허기를 더 빨리 느끼게 됩니다. 이는 과식으로 이어지는 악순환의 시작입니다.

또한 카페인은 인슐린 감수성을 일시적으로 낮춰, 같은 음식을 먹어도 혈당이 더 높이 오르게 만들 수 있습니다. 공복 시간을 길게 유지하는 것이 다이어트의 핵심인데, 커피는 그 공복 상태를 방해하는 주범 중 하나입니다.

이번 주 미션은 간단합니다. 커피는 하루 딱 1잔, 반드시 식사 후 2시간이 지난 뒤에 마시세요. 블랙커피 또는 설탕·시럽 없는 아메리카노만 허용됩니다. 달달한 카페 음료는 이번 12주 동안 최대한 피해주세요. 작은 습관 하나가 몸의 호르몬 균형을 되찾는 첫걸음이 됩니다. 이번 주는 집중 상담 주간으로, 설문 결과를 바탕으로 인스타그램 DM으로 개별 연락드릴 예정입니다.

#### 2주차 (6/8–6/14): 아침 식단에서 정제 탄수화물 빼기

우리가 매일 아침 먹는 흰 밥, 빵, 시리얼은 빠르게 혈당을 올립니다. 혈당이 급격히 오르면 인슐린이 대량 분비되고, 인슐린은 남은 포도당을 지방으로 전환하여 저장합니다. 이 사이클이 반복되면 몸은 점점 인슐린에 둔감해지고, 결국 지방이 쌓이기 쉬운 체질이 됩니다.

아침은 하루 중 인슐린 감수성이 가장 예민한 시간대입니다. 이 시간에 정제 탄수화물 대신 단백질과 건강한 지방으로 식사를 시작하면, 혈당이 천천히 오르고 오전 내내 포만감이 유지됩니다.

이번 주 추천 아침 식단은 **계란 + 올리브오일 + 아보카도 + 사과**입니다. 계란은 단백질과 지방의 완벽한 조합이고, 아보카도의 불포화지방산은 포만감을 높이며, 사과는 식이섬유가 풍부해 혈당 상승을 완만하게 해줍니다. 단 1주일만 실천해도 오전에 간식이 당기는 횟수가 눈에 띄게 줄어드는 것을 느낄 수 있을 거예요.

#### 3주차 (6/15–6/21): 식후 10분 안에 10분 걷기

식사 후 혈당은 30~60분 사이에 가장 높이 올라갑니다. 이때 가만히 앉아 있으면 인슐린이 대량 분비되어 포도당을 지방으로 저장합니다. 하지만 이 시간에 가볍게 걷기만 해도 근육이 혈당을 직접 소모하기 시작해, 인슐린 분비량을 크게 줄일 수 있습니다.

연구에 따르면 식후 10분 걷기는 30분 동안 가만히 있는 것과 비교해 식후 혈당 상승을 최대 30% 낮출 수 있습니다. 격렬한 운동이 아니어도 됩니다. 집 안에서 왔다 갔다 해도, 계단을 오르내려도 효과가 있습니다.

미션은 매 식사 후 10분 이내에 최소 10분간 걷는 것입니다. 하루 세 끼라면 하루 30분, 부담 없이 실천할 수 있는 운동량입니다. 1주일 후에는 식곤증이 줄어들고 오후 에너지가 올라가는 것을 직접 체감하게 될 거예요. 이 습관 하나가 몸이 지방을 태우는 체질로 바뀌는 강력한 스위치가 됩니다.

#### 4주차 (6/22–6/28): 야식 금지 + 7시간 이상 수면

밤 9시 이후에 먹는 음식은 낮에 먹는 같은 음식보다 지방으로 저장될 가능성이 훨씬 높습니다. 저녁 이후에는 몸의 신진대사가 낮아지고, 인슐린 감수성도 떨어지기 때문입니다. 특히 술은 간이 지방 대사 대신 알코올 분해에 집중하게 만들어, 그날 섭취한 지방이 거의 그대로 체지방으로 쌓이게 됩니다.

수면은 다이어트의 숨겨진 핵심입니다. 수면이 부족하면 식욕 억제 호르몬인 렙틴은 줄고, 식욕 촉진 호르몬인 그렐린은 증가합니다. 7시간 미만으로 자는 사람은 충분히 자는 사람보다 비만 위험이 최대 2배 높다는 연구 결과도 있습니다.

이번 주 미션은 두 가지입니다. 첫째, 저녁 9시 이후 야식 및 음주 절대 금지. 둘째, 매일 7시간 이상 수면. 잠자리에 드는 시간을 30분만 앞당겨보세요. 수면의 질이 올라가면 다음 날 식욕이 자연스럽게 줄어들고, 다이어트 전체가 훨씬 수월해집니다.

#### 5주차 (6/29–7/5): 간식·초가공식품·액상과당·탄산음료 없애기

편의점 과자, 음료수, 탄산음료 속에는 '액상과당(HFCS)'이 숨어 있습니다. 액상과당은 일반 설탕보다 단맛이 강하고, 포만감 없이 칼로리만 올립니다. 더 큰 문제는 액상과당이 간에서 직접 지방으로 전환된다는 점입니다. 혈당을 올리지 않아 '건강한 것처럼' 보이지만, 사실은 내장지방을 가장 빠르게 쌓는 주범입니다.

초가공식품(과자, 라면, 냉동식품, 소시지 등)은 먹을수록 더 먹고 싶게 설계된 식품입니다. 뇌의 보상 회로를 자극해 중독성을 유발하고, 실제 포만감은 거의 없어 과식하게 만듭니다.

이번 주는 이 모든 것을 끊는 주입니다. 탄산음료는 제로 음료 포함 가급적 피하고, 간식이 당길 때는 견과류 한 줌, 삶은 달걀, 치즈 한 조각으로 대체하세요. 5주차에 접어들면 몸이 '진짜 배고픔'과 '습관적 허기'를 구분하기 시작합니다. 이번 주가 그 전환점이 될 거예요.

#### 6주차 (7/6–7/12): 양치할 때 스쿼트 15회 3세트 (운동 스낵)

'운동 스낵'이란 하루에 짧게 여러 번 나눠 운동하는 방식입니다. 30분 헬스장 운동보다 하루 3번 10분씩 움직이는 것이 혈당 조절과 근육 유지에 더 효과적이라는 연구 결과가 늘고 있습니다. 특히 바쁜 일상에서 '운동할 시간이 없다'는 핑계를 없애주는 방법이기도 합니다.

스쿼트는 허벅지, 엉덩이, 코어 근육을 동시에 자극하는 최고의 하체 운동입니다. 큰 근육일수록 에너지 소모량이 많고, 기초대사량을 높여 쉬는 동안에도 지방이 타게 만듭니다. 하루 두 번 양치(아침+저녁)에 스쿼트 15회 3세트를 붙이면, 총 90회, 매일 자연스럽게 운동이 됩니다.

처음에는 의자를 잡고 해도 괜찮습니다. 무릎이 발끝을 넘지 않도록 주의하며, 엉덩이를 뒤로 빼면서 천천히 앉았다 일어나세요. 1주일 후 허벅지 앞쪽에 묵직한 자극이 느껴진다면 제대로 하고 있는 겁니다.

#### 7주차 (7/13–7/19): 식후 10분 안에 계단 오르기 10분

3주차에 배운 식후 걷기를 업그레이드할 시간입니다. 계단 오르기는 같은 시간 걷기보다 칼로리 소모가 약 3배 높고, 하체 근력과 심폐 기능을 동시에 향상시킵니다. 식후에 계단을 오르면 혈당이 근육으로 빠르게 흡수되어 인슐린 분비가 줄어들고, 체지방 연소 환경이 만들어집니다.

엘리베이터 대신 계단을 선택하는 것만으로도 하루 활동량이 눈에 띄게 늘어납니다. 직장이나 아파트에 살고 계신다면, 점심 식사 후 계단을 10분만 오르내려 보세요. 숨이 약간 차는 정도면 딱 좋은 강도입니다.

무릎이 불편하신 분은 경사로 걷기나 실내 자전거로 대체해도 됩니다. 중요한 것은 '식후 10분 안에 움직이기'라는 타이밍입니다. 이 타이밍이 혈당 스파이크를 막는 가장 결정적인 순간입니다.

#### 8주차 (7/20–7/26): 주 2회 유산소+근력 운동

지금까지 7주 동안 식습관과 가벼운 운동 습관을 만들었습니다. 이제 본격적으로 체성분을 바꿀 시간입니다. 근육은 지방보다 기초대사량을 훨씬 많이 소모합니다. 근육량이 늘면 아무것도 안 해도 더 많은 칼로리가 소모되는 몸이 됩니다.

이번 주 미션은 주 2회, 유산소 운동 20분 + 근력 운동 20분을 함께 하는 것입니다. 헬스장이 부담스럽다면 집에서 유튜브 운동 영상을 따라해도 충분합니다. 유산소는 빠르게 걷기, 자전거, 줄넘기 등이 좋고, 근력은 스쿼트, 런지, 푸시업 등 도구 없이 할 수 있는 맨몸 운동을 추천합니다.

운동 후 30분 이내에 단백질(계란, 닭가슴살, 그릭요거트 등)을 섭취하면 근육 회복과 성장에 도움이 됩니다. 힘들더라도 주 2회만 지켜주세요. 마지막 4주를 향해 가는 중요한 전환점입니다.

#### 9주차 (7/27–8/2): 보상성 음식 절대 금지 — 취미로 보상하기

"오늘 힘들었으니까 맛있는 거 먹어야지." 이 생각, 한 번쯤 해본 적 있으신가요? 스트레스를 음식으로 해소하는 패턴은 뇌의 도파민 보상 회로가 음식과 연결된 결과입니다. 이 연결을 끊지 않으면 다이어트가 끝난 후에도 요요가 반복됩니다.

이번 주 미션은 스트레스를 받거나 수고한 나에게 음식 대신 다른 방식으로 보상하는 연습입니다. 좋아하는 음악 듣기, 반신욕, 산책, 좋아하는 드라마 한 편, 친구와 통화 등 무엇이든 괜찮습니다. 자신만의 비음식 보상 목록을 3가지 이상 정해두고, 충동이 올 때 그것을 실천해 보세요.

음식으로 채우려 했던 감정의 빈자리를 다른 것으로 채우는 연습. 처음에는 어색하고 부족하게 느껴질 수 있습니다. 하지만 이 훈련이 12주 이후에도 체중을 유지하게 해주는 진짜 힘이 됩니다.

#### 10주차 (8/3–8/9): 식사 중 영상 금지 — 20~30분 천천히 먹기

밥 먹으면서 유튜브, 넷플릭스, 스마트폰 보시나요? 영상에 집중하면 뇌가 먹는 것을 제대로 인식하지 못합니다. 포만감을 느끼게 해주는 렙틴 신호가 뇌에 전달되려면 최소 15~20분이 필요한데, 빠르게 먹으면 신호가 오기도 전에 과식하게 됩니다.

이번 주 미션은 식사 중 TV, 스마트폰, 유튜브 등 모든 영상을 끄는 것입니다. 대신 음식의 맛, 식감, 향에 집중하며 20~30분에 걸쳐 천천히 드세요. 음식의 양은 줄이지 않아도 됩니다. 단지 속도만 늦추세요.

처음에는 어색하고 심심하게 느껴질 수 있습니다. 하지만 1주일만 지나면 같은 양을 먹어도 더 배부르게 느껴지는 경험을 하게 됩니다. 뇌와 위장이 다시 대화하기 시작하는 것입니다. 식사를 즐기는 것 자체가 다이어트가 될 수 있습니다.

#### 11주차 (8/10–8/16): 식사 전 채소(수용성 식이섬유) 먼저 먹기

'채소 먼저' 식사법은 혈당 조절에 있어 가장 강력한 식이 전략 중 하나입니다. 채소에 풍부한 수용성 식이섬유는 위장에서 젤 형태로 변해 음식의 흡수 속도를 늦춥니다. 이 덕분에 같은 식사를 해도 혈당이 훨씬 완만하게 오르고, 인슐린 분비량이 줄어듭니다.

미션은 간단합니다. 밥, 국, 반찬을 먹기 전에 채소 반찬이나 샐러드를 먼저 드세요. 최소 5분 이상 채소만 먹은 후 다른 음식을 시작하면 효과가 더 좋습니다. 오이, 양상추, 브로콜리, 파프리카, 당근 등 어떤 채소든 괜찮습니다.

11주차까지 오신 여러분은 이미 많은 것이 바뀌어 있을 겁니다. 이 마지막 식이 습관 하나를 추가하면, 12주 이후 일상에서도 자연스럽게 혈당을 관리하는 몸이 완성됩니다.

#### 12주차 (8/17–8/23): 매 끼니 단백질 20~30g + 건강한 지방으로 식사 시작

12주 여정의 마지막 미션입니다. 단백질은 포만감을 가장 오래 유지시켜 주는 영양소입니다. 식사의 첫 번째로 단백질과 건강한 지방을 먹으면, 혈당 상승이 억제되고 이후 탄수화물 흡수 속도도 함께 느려집니다.

매 끼니 단백질 20~30g은 계란 3개, 닭가슴살 100g, 두부 150g, 그릭요거트 200g 정도에 해당합니다. 여기에 올리브오일 1스푼, 아보카도 반 개, 견과류 한 줌 등 건강한 지방을 함께 곁들여 식사를 시작하세요.

이 습관은 12주 캠프가 끝난 이후에도 평생 유지할 수 있는 가장 강력한 식이 전략입니다. 지금까지 12주 동안 정말 잘 해오셨습니다. 마지막 주도 완주하고, 8월 31일 최종 결과지 제출로 멋진 마무리를 함께해요!

## 개발 시 주의사항

1. **한글 입력:** 텍스트 입력은 반드시 `KoreanInput` 컴포넌트 사용 (IME 조합 버그)
2. **캠프 시작일 변경:** `src/lib/missions.ts`의 `CAMP_START_DATE` 하나만 수정
3. **알림톡 시작일 변경:** `src/lib/cron-alimtalk.ts`의 날짜 수정
4. **recharts SSR:** 차트는 `mounted` state 후에만 렌더링 (`{mounted && <Chart />}`)
5. **Supabase 초기화:** `supabase.ts`는 Proxy 패턴 — 모듈 레벨에서 직접 초기화 금지
6. **API 클라이언트:** 모듈 레벨 생성 금지, 함수 내부에서 생성 (`alimtalk.ts` 참고)
7. **weeklyRecords:** localStorage에서 로드 시 undefined 가능 → `data.weeklyRecords ?? []` 필수
8. **KST 날짜:** `new Date(Date.now() + 9*60*60*1000).toISOString().slice(0,10)` 사용
9. **배포:** `vercel --prod` (또는 git push → Vercel 자동 배포)
